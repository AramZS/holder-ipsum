holder-ipsum.js Fill your pages with lorem-ipsum without copy-and-pasting or writing any Javascript.

I've got plenty of lorem-ipsum what are the benefits of holder-ipsum.js?

Doesn't require copying and pasting from generator sites
Every time you need some lorem-ipsum, you don't need to switch to another tab or application to make you the right amount of text; holder-ipsum.js builds the words in the browser as soon as the page loads.
Easy-to-update
Changed your mind about how much filler text, you want? With a couple of key strokes you can see the changes in your page in seconds.
Switching between a number or words, sentences, or paragraphs is just as easy.
It's flexible
If you need to see what a list is going to look like, you can rapidly generate child elements such as <li>.
Control over children follows the same syntax as the parent.
Don't like altering your markup to get some lorem-ipsum?
No problem! holder-ipsum.js exposes the entire interface as a public API so that you can exercise full control from Javascript code if you choose.

Getting started

  1. Grab a copy of holder-ipsum.js.
  2. Include holder-ipsum.js just before the closing </body> tag.
  3. Mark-up the all the elements you want to be filled with lorem-ipsum.
  4. Reload the page and see the results.

Syntax

All the generation of lorum ipsum is targeted from data attributes on the elements you want to fill with placeholder text. The script looks for elements with data-holder-ipsum-mode declared on them.

There are 5 modes of operation of holder-ipsum which are controlled by the data-holder-ipsum-mode. These are:

  • words
  • sentence
  • paragraph
  • paragraphs
  • children

Each mode can take additional parameters in the form of data attributes namespaced to that particular mode. For more detail see the discussion of each mode.

words

Declaring data-holder-ipsum-mode="words" will fill the specified element with words. This mode requires that you pass the number of words to display using data-holder-ipsum-words-count.

Warning: you must specify an integer for data-holder-ipsum-words-count. There is no exception handling around this parameter.
Warning: there is an upper limit to the number of words you can generate (currently 182) due to the size of the lorem-ipsum dictionary. For larger bodies of text use, e.g. sentence or paragraph.

Additionally you can choose whether to have the common words from lorem-ipsum at the beginning of your words. This is controlled by the optional attribute data-holder-ipsum-words-common which must be either true or false. By default, this is false.

Example

<p
  data-holder-ipsum-mode="words"
  data-holder-ipsum-words-count="10"
  data-holder-ipsum-words-common="false"
><!-- 10 words will appear here --></p>

sentence

If you want a whole sentence of lorem-ipsum, use data-holder-ipsum-mode="sentence". In sentence mode, no additional parameters are expected and anything else passed will be ignored.

Hint: There is quite a bit of variability in the length of the sentences generated due to the random nature of the algorithm. For more predictable blocks of text, you may be better off using words mode.

Example

<p
  data-holder-ipsum-mode="sentence"
><!-- sentence will appear here --></p>

paragraph

To output a paragraph of lorem-ipsum, use data-holder-ipsum-mode="paragraph". In paragraph mode, no additional parameters are expected and anything else passed will be ignored.

Example

<p
  data-holder-ipsum-mode="paragraph"
><!-- paragraph will appear here --></p>

paragraphs

For large blocks of text, data-holder-ipsum-mode="paragraphs" will come in handy. This outputs the number of paragraphs you specify with data-holder-ipsum-paragraphs-count.

Additionally, you can control whether the first paragraph begins with the common lorem ipsum phrase with the data-holder-ipsum-paragraphs-common attribute.

Note: the paragraphs will be joined with two <br> elements. For finer control of this behaviour see the children mode.

Example

<p
  data-holder-ipsum-mode="paragraphs"
  data-holder-ipsum-paragraphs-count="4"
  data-holder-ipsum-paragraphs-common="true"
><!-- 4 paragraphs will appear here, starting with the common phrase --></p>

children

If you're experimenting with how a <ol> might look and you want to vary the number of <li> which are subordinate to it, use, data-holder-ipsum-mode="children".

This is the most powerful mode of holder-ipsum.js and its parameters fall into two discrete categories:

Those which control the beahviour of the generation of the children
data-holder-ipsum-children-count
data-holder-ipsum-children-element
data-holder-ipsum-children-mode
Those which control the mode of the children
e.g. data-holder-ipsum-children-words-count if
data-holder-ipsum-children-mode="words"

To control the number of children that will be generated set the data-holder-ipsum-children-count. This must be an integer. Default: 3

To set the type of tag used to build the children, set data-holder-ipsum-children-element, e.g. data-holder-ipsum-children-element="dd". NB This should not contain the angular brackets – just the name of the tag. Default: p

To specify which mode should be used to generate the children, set data-holder-ipsum-children-mode. This can be any value which is also valid for data-holder-ipsum-mode.

To pass arguments to the generator for the children, ensure that the argument is namespaced, e.g. data-holder-ipsum-children-words-count.

Example

<ol
  data-holder-ipsum-mode="children"
  data-holder-ipsum-children-mode="words"
  data-holder-ipsum-children-element="li"
  data-holder-ipsum-children-count="10"
  data-holder-ipsum-children-words-count="10"
><!-- 10 li elements will be appended here, each with 10 random words --></ol>

Public API

All the generator functions are exposed as a public API on the global HolderIpsum object. The methods are:

  • words
  • sentence
  • paragraph
  • paragraphs

You will recognize these from the declarative form of use of holder-ipsum.js. However, there are some subtle differences which are documented below.

words

Usage: HolderIpsum.words(word_count, suppress_common)

word_count
The number of unique words to generate
suppress_common
Whether to suppress the common phrase from lorem-ipsum. This is an optional parameter defaulting to false.
Warning: there is an upper limit to the number of words you can generate (currently 182) due to the size of the lorem-ipsum dictionary. For larger bodies of text use, e.g. sentence or paragraph.

Example

// Generate 40 words of lorem-ipsum
// with a random beginning
var words = HolderIpsum.words(40, true);

// Generate 35 words of lorem-ipsum
// starting with the common phrase
var ipsum_words = HolderIpsum.words(35, false);

sentence

Usage: HolderIpsum.sentence()

There are no parameters for this function.

Example

// Generate a sentence of lorem-ipsum
var sentence = HolderIpsum.sentence();

paragraph

Usage: HolderIpsum.paragraph()

There are no parameters for this function.

Example

// Generate a paragraph of lorem-ipsum
var paragraph = HolderIpsum.paragraph();

paragraphs

Usage: HolderIpsum.paragraphs(paragraph_count, suppress_common)

paragraph_count
The number of paragraphs to generate
suppress_common
Whether to suppress the common phrase from lorem-ipsum. This is an optional parameter defaulting to false.

This function returns an array of strings.

Example

// Generate 3 paragraphs of lorem-ipsum
// starting with the random words
var paragraphs = HolderIpsum.paragraphs(3, true);

// Generate 4 paragraphs of lorem-ipsum
// starting with the common phrase
var paragraphs = HolderIpsum.paragraphs(4, false);

The code uses all native Javascript and should be supported by all modern broswers. It has been tested in:

  • Safari 5.0.6
  • Firefox 15.0
